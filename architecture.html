🔒 Smart Contract Security Analysis Pipeline

Single-Contract LLM-Powered Vulnerability Deduplication

📄 Input Layer
Solidity Contract (.sol)
Source code file and vulnerability category as command-line arguments
↓
🎯 Pipeline Orchestrator (run_pipeline.py) CORE IMPLEMENTATION
Step 1: Solc Detection
Parses pragma to detect Solidity version (0.4.x–0.8.x) and switches solc
Step 2: Slither Analysis
Runs Slither static analyser from .venv, captures JSON output (detectors array)
Step 3: Mythril via Docker
Launches Docker container with solc binary volume mount, captures JSON (issues array)
↓
🔄 Step 4: Parse & Combine Findings IMPLEMENTED
Parser (parse_slither, parse_mythril)
Converts Slither/Mythril JSON to unified Finding objects
Combiner (combine_and_save)
Merges findings into combined_<contract>.json in data/combined/
↓
🤖 Step 5: LLM Deduplication & P-Category Injection YOUR CORE WORK
Azure OpenAI GPT-4
Semantic analysis to identify and group duplicate vulnerability alerts
SWC Mapper (llm_deduplicator.py)
Reads configs/p_mapping.yaml, maps SWC IDs → P-keys (P1–P15), injects ethtrust_rule
Result Enrichment
Each unique vulnerability tagged with SWC ID, severity, P-category, and token usage
↓
📊 Steps 6–8: Results & Reporting IMPLEMENTED
result.json
Final report: raw alerts, unique vulns, reduction %, cost, deduplicated findings
summary.csv
Single-row append per run; tracks contract, category, metrics across batches
Timestamped Logging (run.log)
Full pipeline trace: binary paths, return codes, file sizes, error messages
↓
✅ Output Layer
Deduplicated JSON Report
Saved to data/deduplicated/<contract>_deduplicated.json with unique findings + P-categories
